# EWWM v0.5.0 Roadmap: Daily-Driver Desktop + VR Hardware

> **Status (2026-03-10)**: v0.5.0 released and tagged. Phases 1-2 (Wayland
> protocols, hardware enablement) were the core release scope. Phase 3-5
> (VR patterns, eye tracking, daily-driver polish) continue as post-release
> work on `feature/v0.5.0-vr-renderer` and linux-xr kernel infrastructure.

## Executive Summary

EWWM has deeply unique VR/biometric capabilities that no other project matches. However,
**9 standard Wayland protocols** are missing that every competitor considers table stakes.
This plan covers the path from VR WM to daily-driver DE, hardware enablement for the honey
server (RX 9070 + Bigscreen Beyond 2e), and implementable VR patterns from lightofmysoul's
KDE research.

**Minimum time to daily-driver: ~4-6 weeks** (60% Rust, 40% Elisp/packaging).

## Phase 1: Missing Wayland Protocols (P0 — Blocking)

9 missing protocols; 7 have Smithay built-in support, 2 need `wayland_protocols_wlr`.

| Protocol | Smithay Module | Effort | Why Blocking |
|----------|---------------|--------|-------------|
| ext-session-lock-v1 | `session_lock` | 2d | Cannot lock screen |
| ext-idle-notify-v1 | `idle_notify` | 1d | No auto-lock on idle |
| zwp-idle-inhibit-v1 | `idle_inhibit` | 0.5d | Lock triggers during video |
| zwp-primary-selection-v1 | `primary_selection` | 0.5d | Middle-click paste broken |
| wlr-data-control-v1 | manual (wlr crate) | 2d | Clipboard dies with app |
| linux-dmabuf-v1 | `dmabuf` | 1d | GPU buffer sharing |
| wp-cursor-shape-v1 | `cursor_shape` | 1d | Cursor missing for apps |
| DPMS (output power) | output API | 0.5d | Monitors can't sleep |
| xdg-activation-v1 | `xdg_activation` | 1d | Launched apps don't focus |

**Elisp companion work:**
- `ewwm-autostart.el` — XDG autostart spec (read .desktop files) — 1d
- `ewwm-session-logout` / `ewwm-session-shutdown` — 0.5d
- Standardize `XDG_CURRENT_DESKTOP` to one value

**Total Phase 1: ~11 days**

## Phase 3: VR Implementable Patterns (from lightofmysoul research)

### Critical

| Pattern | Implementation | Effort |
|---------|---------------|--------|
| **Follow Mode** | `FollowConfig` in vr_interaction.rs — start/stop FOV thresholds, grab-all, recenter. `ewwm-vr-follow.el` with defcustoms. | 3d |
| **XWayland Focus Routing** | Force gaze-selected surface focus via Seat API. UV→pixel coord remapping. | 2d |

### High

| Pattern | Implementation | Effort |
|---------|---------------|--------|
| **Pointer Lock Per-Window** | `PointerConstraintsHandler` — deny lock in VR mode unless allowed per-surface. `allow_pointer_lock` field in SurfaceData. | 2d |
| **Transient Chains in 3D** | Compute popup 3D position from parent plane + offset. Recursive chains with Z-depth. | 2d |
| **Passthrough / AR** | Configurable `EnvironmentBlendMode` (currently hardcoded OPAQUE). Query supported modes at session start. | 1d |
| **XR_EXTX_overlay** | Overlay session creation with `XrSessionCreateInfoOverlayEXTX`. Alpha compositing. Monado supports this. openxrs 0.21 has types. | 3d |
| **Spatial Anchors** | `XR_EXT_spatial_anchor` + persistence. Pin windows to physical positions across sessions. | 3-5d |

### Medium / Low

| Pattern | Priority | Effort |
|---------|----------|--------|
| Leasable Output Flag | Medium | 2d |
| Radial Menu | Medium | 3d |
| Screen Capture Visibility | Low | 3d |

**Total Phase 3: ~20-25 days (can be parallelized with Phase 1-2)**

## Phase 4: Honey Server Hardware Enablement

### AMD RX 9070 (RDNA 4 / Navi 48 / gfx1201)

**Requirements:**
- Kernel >= 6.13 (RDNA 4 enabled by default; < 6.13 needs `amdgpu.exp_hw_support=1`)
- Mesa >= 25.0 (RADV — sole Vulkan ICD; AMDVLK discontinued Sep 2025)
- Firmware: upstream `linux-firmware.git` (navi48_* blobs)
- Known issue: kernel 6.18.x ROCm crashes — pin to 6.17.x or wait for fix
- VR power: set `power_dpm_force_performance_level=high` during VR session

**NixOS config:**
```nix
hardware.graphics.enable = true;
hardware.graphics.enable32Bit = true;
hardware.firmware = [ pkgs.linux-firmware ];
environment.variables.AMD_VULKAN_ICD = "RADV";
boot.kernelPackages = pkgs.linuxPackages_latest;
boot.extraModprobeConfig = "options amdgpu ppfeaturemask=0xffffffff";
```

### Bigscreen Beyond 2e

**Connection:** Fiber optic cable → Link Box → 1x DP 1.4 + 2x USB 3.0
**USB:** VID `0x35BD`, PID `0x0101` (Bigscreen, Inc.)
**HID:** `/dev/hidrawN` for display mode switching (via beyond_squared Rust utility)
**IPD:** Mechanical (53-70mm), not motorized — no HID control
**Eye tracking:** Cloud calibration pipeline → local GPU model → OpenXR `XR_EXT_eye_gaze_interaction`

**DRM non_desktop:** Kernel patch NOT merged upstream. Workarounds:
1. `video=DP-3:d` kernel cmdline
2. `xrandr --output DP-3 --set non-desktop 1` at runtime
3. Custom kernel build with EDID quirk patch
4. Compositor-level filter by EDID manufacturer `BIG`

**Monado:** Use Matrix's fork for Beyond-specific patches. nixpkgs-xr overlay for packaging.

**udev rule (add to 99-exwm-vr.rules):**
```
KERNEL=="hidraw*", SUBSYSTEM=="hidraw", ATTRS{idVendor}=="35bd", ATTRS{idProduct}=="0101", MODE="0660", TAG+="uaccess"
SUBSYSTEM=="usb", ATTR{idVendor}=="35bd", MODE="0666"
```

### Full NixOS VR Configuration

```nix
{
  hardware.graphics = { enable = true; enable32Bit = true; };
  hardware.firmware = [ pkgs.linux-firmware ];
  environment.variables.AMD_VULKAN_ICD = "RADV";
  boot.kernelPackages = pkgs.linuxPackages_latest;
  boot.extraModprobeConfig = "options amdgpu ppfeaturemask=0xffffffff";

  # Monado via nixpkgs-xr
  services.monado = { enable = true; defaultRuntime = true; };
  systemd.user.services.monado.environment = {
    STEAMVR_LH_ENABLE = "1";
    XRT_COMPOSITOR_COMPUTE = "1";
    XRT_COMPOSITOR_DESIRED_MODE = "1";  # 90 Hz
  };

  users.users.jsullivan2.extraGroups = [ "video" "render" "input" ];
  security.pam.loginLimits = [
    { domain = "@video"; type = "-"; item = "rtprio";  value = "99"; }
    { domain = "@video"; type = "-"; item = "nice";    value = "-20"; }
    { domain = "@video"; type = "-"; item = "memlock"; value = "unlimited"; }
  ];

  services.udev.extraRules = ''
    KERNEL=="hidraw*", SUBSYSTEM=="hidraw", ATTRS{idVendor}=="35bd", ATTRS{idProduct}=="0101", MODE="0660", TAG+="uaccess"
    SUBSYSTEM=="usb", ATTR{idVendor}=="35bd", MODE="0666"
  '';

  programs.steam.enable = true;
}
```

## Phase 5: Configuration Strategy

**Verdict: Do NOT adopt Dhall.** `serde_dhall` is unmaintained, Dhall can't express behavioral
Elisp config, and no one configures Emacs with Dhall.

**Strategy instead:**
1. **NixOS users** (primary): Expand existing Nix-to-Elisp pipeline in `nix/home-manager/exwm-vr.nix`. Already works. Add more typed options.
2. **Non-Nix users**: Add a JSON/TOML config file for the compositor with JSON Schema validation. Emacs reads JSON natively.
3. **Future type safety**: Evaluate **Nickel** (Rust-native, Tweag-backed, Nix-evolution) if cross-target config becomes critical.
4. **Keep Elisp** as the primary config for WM behavior (hooks, keymaps, manage rules).
